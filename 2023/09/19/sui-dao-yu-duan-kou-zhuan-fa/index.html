1.前置知识

1.1 IP寻址与网卡

在网络上通信的每台计算机都需要一个 IP 地址。如果没有，则不通网。IP 地址可以在软件中分配，通常从 DHCP 服务器自动获取，但也有静态分配的IP地址。
静态分配的 IP 地址：

  • Servers 服务器
  • Routers 路由器
  • Switch virtual interfaces 交换机虚拟接口
  • Printers 打印机
  • And any devices that are providing critical services to the network
    以及为网络提供关键服务的任何设备

IP地址无论是动态的还是静态的都将被分配给网卡(NIC)，一台计算机可以有多个网卡，可以是物理的，也可以是虚拟的，即一台计算机可以分配多个IP地址。

当数据包在离开计算机之前创建并具有目标时，路由表用于决定将其发送到何处。例如，如果我们尝试连接到目标 10.129.0.0 ，我们可以从路由表中判断数据包将发送到哪里以到达那里。它将被转发到 Gateway 相应的网卡 （ Iface ）。它通过自己的直接连接的接口（eth0，eth1，tun0）解析这些路由。指定为路由器的独立设备通常将使用静态路由创建、动态路由协议和直接连接的接口的组合来学习路由。发往路由表中不存在的网络的任何流量都将发送到 default route 该网关也称为默认网关或最后手段网关。

2.使用 SSH 和 SOCKS 隧道进行动态端口转发

2.1 介绍

端口转发是一种允许我们将通信请求从一个端口重定向到另一个端口的技术。端口转发使用 TCP 作为主要通信层，为转发的端口提供交互式通信。但是，可以使用不同的应用层协议，例如SSH甚至SOCKS（非应用层）来封装转发的流量。（传输层）

攻击机：10.10.15.5
Ubuntu服务器：10.129.15.50（已拿到passwd，目标服务器）

Nmap 输出显示 SSH 端口已打开。但是mysql的端口显示closed，说明我们现在无法直接直接访问到。
要访问 MySQL 服务，
①我们可以通过 SSH 连接到服务器并从 Ubuntu 服务器内部访问 MySQL
②我们可以将其映射到端口 1234 上的本地主机并在本地访问它
在本地访问它的一个好处是，如果我们想在 MySQL 服务上执行远程漏洞利用，如果没有端口转发，我们将无法做到这一点。这是由于MySQL本地托管在端口 3306 上的Ubuntu服务器上，我么无法直接访问。因此，我们将使用以下命令通过SSH将我们的本地端口（1234）转发到Ubuntu服务器。

ssh -L [本地端口]:[目标主机]:[目标端口] [用户名]@[远程服务器]

2.2.6 转发多个端口

同样，如果我们想将多个端口从 Ubuntu 服务器转发到您的本地主机，您可以通过在 ssh 命令中包含 local port:server:port 参数来实现。例如，以下命令将 apache Web 服务器的端口 80 转发到攻击主机的本地端口 上 8080 。

MengTL4@htb[/htb]$ ssh -L 1234:localhost:3306 8080:localhost:80 ubuntu@10.129.202.64

在当前场景下，我们通过ssh连接到目标服务器上了，与先前的情景不同，我们不知道网络的另一侧有哪些服务。因此，我们可以扫描网络上较小范围的 IP（172.16.5.1-200）或整个子网（172.16.5.0/23即172.16.5.0-172.16.6.255）(CIDR表示法)。我们无法直接从攻击主机执行此扫描，因为它没有路由到172.16.5.0/23。为了实现这一点，我们需要执行动态端口转发，并通过Ubuntu服务器将我们的网络数据包进行中转。我们可以通过在我们的本地主机上启动一个SOCKS监听器，然后配置SSH，在连接到目标主机后通过SSH将流量转发到网络（172.16.5.0/23）来完成这一操作。
这被称为SOCKS代理的SSH隧道

我们有一个 172.16.5.0/23 的 NAT 网络(网络地址转换)，我们无法直接访问该网络。
在上图中，攻击主机启动 SSH 客户端并请求 SSH 服务器允许它通过 ssh 套接字发送一些 TCP 数据。SSH 服务器以确认响应，然后 SSH 客户端开始侦听 localhost:9050 。您在此处发送的任何数据都将通过SSH广播到整个网络（172.16.5.0 / 23）。我们可以使用以下命令来执行此动态端口转发。

动态端口转发本质上是把本地主机A上运行的SSH客户端转变成一个SOCKS代理服务器。

2.3.2 使用 SSH 启用动态端口转发

MengTL4@htb[/htb]$ ssh -D 9050 ubuntu@10.129.202.64

“-D” 参数请求SSH服务器启用动态端口转发。一旦启用了这个功能，我们将需要一个工具，可以将任何工具的数据包路由到端口9050上。我们可以使用工具proxychains来实现这一点，它能够通过TOR、SOCKS和HTTP/HTTPS代理服务器重定向TCP连接，还允许我们将多个代理服务器串联在一起。使用proxychains，我们还可以隐藏请求主机的IP地址，因为接收主机只会看到中转主机的IP地址。Proxychains通常用于强制应用程序的TCP流量经过托管的代理，如SOCKS4/SOCKS5、TOR或HTTP/HTTPS代理。

3.使用 SSH 进行远程/反向端口转发(场景四)

我们已经看过了本地端口转发，其中SSH可以侦听本地主机并将远程主机上的服务转发到我们的端口，以及动态端口转发，我们可以通过跳板主机将数据包发送到远程网络。但有时，我们可能还希望将本地服务转发到远程端口。
在我们之前的情况下，我们可以通过 Ubuntu 服务器移动到到 Windows 主机。
但是如果我们想要获取一个反向shell呢？
Windows主机的出站连接仅限于172.16.5.0/23网络。这是因为Windows主机与攻击主机所在的网络没有直接连接。如果我们在攻击主机上启动Metasploit监听器并尝试获取反向Shell，我们将无法直接建立连接，因为Windows服务器不知道如何将离开其网络（172.16.5.0/23）的流量路由到达10.129.x.x。
在这些情况下，我们需要找到一个中转主机，它是攻击主机和Windows服务器之间的常见连接点。在我们的情况下，我们的中转主机将是Ubuntu服务器，因为它可以连接到我们的攻击主机和Windows目标。
为了在Windows上获得Meterpreter shell，我们将使用msfvenom创建一个Meterpreter HTTPS payload，但负载的反向连接配置将使用Ubuntu服务器的主机IP地址（172.16.5.129）。我们将使用Ubuntu服务器上的端口8080来转发所有反向数据包到我们攻击主机上的8000端口，在那里我们运行着Metasploit监听器。

5.Socat重定向与反向shell

Socat是一种双向中继工具，可以在不需要使用SSH隧道的情况下，在两个独立的网络通道之间创建管道套接字。它充当了一个重定向器的角色，可以在一个主机和端口上监听数据，并将数据转发到另一个IP地址和端口。我们可以使用与上一节提到的相同命令在攻击主机上启动Metasploit的监听器，并在Ubuntu服务器上启动socat。

5.1 启动Socat监听器

ubuntu@Webserver:~$ socat TCP4-LISTEN:8080,fork TCP4:10.10.14.18:80
